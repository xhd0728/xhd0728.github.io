```shell
# install
bundle install

# local preview only
bundle exec jekyll serve

# production build
JEKYLL_ENV=production bundle exec jekyll build
```

Do not deploy the `_site` output generated by `jekyll serve`. Use the production build or the GitHub Actions workflow output instead.

Production deployment is handled by GitHub Actions:

- Push to `main`
- Workflow builds the site into `_site`
- Workflow force-pushes the built output to the `cloudflare-pages` branch
- Cloudflare Pages should use `cloudflare-pages` as the production branch

Recommended Cloudflare Pages settings for this repo:

- Production branch: `cloudflare-pages`
- Preview branch control: `None`
- Build command: leave blank
- Build output directory: `.`
- Custom domain: `https://xinhaidong.top`
